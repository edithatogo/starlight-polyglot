The TypeScript handler uses [TypeDoc](https://typedoc.org/) programmatically (not as a subprocess) to extract API documentation from TypeScript source code. TypeDoc converts TypeScript type information and JSDoc comments into a reflection tree, which is then transformed into Starlight MDX pages.

## Prerequisites

- **TypeDoc** >= 0.28.0: `npm install typedoc typedoc-plugin-markdown`

TypeDoc and `typedoc-plugin-markdown` are optional peer dependencies of starlight-polyglot — you only need to install them if you use the TypeScript handler.

## Configuration

```js
polyglot({
  typescript: {
    // Required: One or more TypeScript entry files
    entryPoints: ['src/index.ts', 'src/core.ts'],

    // Optional: Path to tsconfig.json for TypeDoc configuration
    tsconfig: 'tsconfig.json',

    // Optional: Output subdirectory (default: "api/typescript")
    output: 'api/typescript',

    // Optional: Include pagination links between pages
    pagination: false,
  },
})
```

### Options

| Option         | Type        | Required | Default            | Description                                   |
|----------------|-------------|----------|--------------------|-----------------------------------------------|
| `entryPoints`  | `string[]`  | Yes      | —                  | TypeScript entry file paths                   |
| `tsconfig`     | `string`    | No       | TypeDoc defaults   | Path to TypeScript configuration file         |
| `output`       | `string`    | No       | `"api/typescript"` | Output subdirectory under `src/content/docs/` |
| `pagination`   | `boolean`   | No       | `false`            | Enable prev/next navigation between pages     |

## How It Works

1. The handler imports TypeDoc's `Application` class directly
2. TypeDoc reads your entry points and tsconfig, converts TypeScript to a reflection tree
3. The handler serializes the reflection tree to JSON
4. Reflections are converted to the shared `ASTModule[]` format with:
   - Modules (kind=1) and Namespaces (kind=2) → ASTModule
   - Classes (kind=128) → ASTClass with methods and properties
   - Functions (kind=64) → ASTFunction with signature and parameters
   - Variables (kind=1024) → ASTVariable with type information

## What Gets Generated

For each module or namespace, the handler generates:

- A **module overview page** listing all classes, functions, and variables
- Individual **class pages** with constructors, methods, properties, and JSDoc
- Individual **function pages** with signatures, parameters, return types, and JSDoc

### TypeDoc Settings Used

The handler configures TypeDoc with:

| Setting                 | Value   | Description                         |
|-------------------------|---------|-------------------------------------|
| `skipErrorChecking`     | `false` | Report TypeScript errors            |
| `excludeExternals`      | `true`  | Exclude external/`node_modules`     |
| `excludePrivate`        | `true`  | Exclude `private` members           |
| `excludeProtected`      | `false` | Include `protected` members         |
| `validation.notExported`| `false` | Don't warn about non-exported types |

## Validation

The handler dynamically imports TypeDoc to verify availability. No separate installation check is needed.

This documentation site uses the TypeScript handler to document the starlight-polyglot plugin's own source code. See the [API Reference](/api/typescript/) section — those pages are generated by starlight-polyglot documenting itself.

## Troubleshooting

### "TypeScript handler requires at least one entryPoint"

Ensure you've provided at least one entry point in the `entryPoints` array.

### "TypeDoc conversion produced no project"

Common causes:

- Entry points don't exist or are not valid TypeScript files
- `tsconfig` path is incorrect
- TypeScript compilation errors (check your code with `tsc --noEmit`)
- Entry points are in `node_modules` and excluded by `skipErrorChecking`

Try running TypeDoc directly to debug:

```bash
npx typedoc src/index.ts --tsconfig tsconfig.json
```

### "typedoc not installed"

Install the peer dependencies:

```bash
npm install typedoc typedoc-plugin-markdown
```

### Exported types not showing up

Ensure your types are **exported** from the entry point files. TypeDoc only documents exported symbols by default. If you need to document non-exported types, contact the maintainers.

### TypeDoc version compatibility

starlight-polyglot requires TypeDoc >= 0.28.0. Check your version:

```bash
npx typedoc --version
```
